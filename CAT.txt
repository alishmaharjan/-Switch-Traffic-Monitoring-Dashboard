#Switch Traffic Monitoring Dashboard
Thruk + Naemon + Python + RRDTool
Project Objective

Develop a lightweight Network Monitoring Dashboard that:

Retrieves interface traffic from Huawei switches
Uses Thruk/Naemon as the monitoring backend
Polls interface data automatically
Stores historical traffic
Generates RRD graphs
Displays graphs inside a custom web dashboard

Phase 1 — Huawei Switch Monitoring
Host Creation

Created Huawei Switch host inside Naemon.

Example

define host{
    host_name       Office-huawei-switch
    address         192.168.200.2
    use             generic-switch
}
Service Creation

Each interface has its own service.

Example

GE0/0/22 Traffic
GE0/0/23 Traffic
GE0/0/24 Traffic
Custom Plugin

Instead of using

check_snmp

a custom plugin

check_iftraffic64.py

was developed.

This plugin:

reads ifHCInOctets
reads ifHCOutOctets
reads ifOperStatus
reads ifSpeed
calculates Mbps
calculates utilization %
caches previous counters
outputs Naemon performance data

Example output

OK - GigabitEthernet0/0/22 UP

RX=23.5 Mbps
TX=5.3 Mbps
RX Util=2.3%
TX Util=0.5%

| rx=23.5 tx=5.3
Phase 2 — Reading Data from Thruk

Instead of polling the switch again, we reuse the monitoring data already collected by Naemon.

The collector reads directly from Livestatus.

Socket

/var/cache/naemon/live

Query

GET services

Columns:
host_name
description
state
plugin_output

Filter:
host_name = Office-huawei-switch

Filter:
description ~~ Traffic

Returned Example

Office-huawei-switch
GE0/0/22 Traffic
0
OK - RX=0.23Mbps TX=0.52Mbps ...

Python parses

interface
state
RX Mbps
TX Mbps
utilization
speed

No additional SNMP polling is required.

Phase 6 — Flask Backend

Flask exposes

GET /api/interfaces

Returns

{
    "summary": {},
    "interfaces": []
}

Dashboard refreshes every

30 seconds

Graph Endpoint

/graph/<interface>/<period>

Example

/graph/GE0_0_22/24h

Backend generates graph if missing.
